一. CNN卷积神经网络

1.1 语义分割-FCN算法详解

1.1.1 整体网络结构

  • 应用背景：2014年CNN在分类任务表现了巨大的潜力，因此某团队就想到把分类任务的CNN用来做语义分割，所以后面很多原理和分类任务相似；

  • 图解网络：以网络图解析FCN的结构；

    • 下采样：下采样负责降低分辨率，扩大感受野，从而能够高效地进行特征提取；从通道来看，图像尺寸由于3*3卷积越来越小，通道越来越多(类似不断放大图像，找到图像的细节)；
    • 上采样：上采样负责恢复分辨率，或者和跳跃连接结合融合底层和高层信息；只有分辨率恢复，才能够对图像上的每个像素点进行类别预测；
    • 输出层：输出层负责对每个像素点进行类别预测；

1.1.2 下采样

  • 应用背景：常规特征提取方法。原FCN论文采用的是VGG网络，但是实际随着深度学习的发展，主干网络大部分情况采用ResNet残差网络；

  • 图解网络：以原理图解析下采样；

    • 32倍采样：FCN网络最高向下采样32倍；

1.1.3 上采样

  • 应用背景：只有分辨率恢复，才能够对图像上的每个像素点进行类别预测；

  • 图解网络：以原理图解析上采样；

    • 多版本上采样：从不同下采样通道截取还原；
    • 转置卷积：如图所示为上采样方法；

1.1.4 输出层

  • 结果解析：输出是21通道的特征图，每个通道都包含这个通道对应类别的概率，哪个概率最高该像素就标记为什么类别；

二. Transformer注意力神经网络

2.1 Transformer算法详解

2.1.1 词嵌入

  • 应用背景：Transformer算法原先应用于自然语言处理领域，故而需要将一段文字或者转换为可被算法识别的数学表达，其表达方式涉及词向量和词嵌入两个概念；

  • 词向量：词向量是将词语直接转为向量，著名的有独热编码，例如下述公式将“我喜欢学习+++++++++”转换为可表示的词向量；

    • 单一表示：每个词都有一位为1的未知维度的列表表示，词语之间的关联性无法通过词向量表达(也就是行业所描述的向量之间没有语义关系)；
    • 维度过大：句子的长度如果过大，输入的维度也将非常大，造成网络参数量和计算量暴增；
    • 新的方法：为解决上述两个问题，Transformer采用词嵌入方法；

  • 词嵌入：词嵌入是实现降维和语义关联的重要方法；

    • 降维：对应词语的词嵌入式计算方式为词向量×嵌入矩阵，例如上述句子的“喜欢”一词可如下式表示；

      • 维度可控：嵌入矩阵W的维度是人为定义的，所有独热编码向量转换后都可以统一维度，原论文取512；
      • 随机初始化：嵌入矩阵内的值和权重w的概念相似，是随机初始化的。模型在一次前向传播后根据反向传播更新梯度时，也会更新嵌入矩阵的参数(也就是嵌入矩阵是可学习的)；

    • 语义关联：正是因为嵌入矩阵是可学习的，所以嵌入矩阵学习后的参数可以让语义相近的词在向量空间上会更接近，出现聚类的一种现象，正如下式中向量空间中“国王”和”王子”的语义更为接近；

2.1.2 绝对位置编码

  • 应用背景：词嵌入只考虑词语之间的关联性，但没有考虑同一个词在不同的位置有不同的含义，所以Transformer添加位置编码方法以标明词语在句子当中的顺序/位置(主流方法有绝对位置编码、相对位置编码和旋转位置编码，Transformer原论文采用的是绝对位置编码)；

  • 数学背景：在计算位置编码前需要先知道正弦/余弦函数的一些基本性质，对于下述公式；

    • 波长越长，频率越低，函数变化就越缓慢；
    • 波长越短，频率越高，函数变化就越快速；

  • 编码公式：对于偶数维度采用正弦函数，对于奇数维度使用余弦函数，正如下式；

    • i较小即低维度时，波长较小，频率较大，位置编码随位置pos的变化较快，对位置变化非常敏感，适用于捕捉短距离信息即局部依赖关系；
    • i较大即高维度时，波长较大，频率较小，位置编码随位置pos的变化较慢，这导致两个位置很远的token在高维度上的编码是相似，在训练过程中注意力会分配相同的权重，从而能够捕获长距离信息即全局依赖关系；
    • 一个词向量的信息分散在每个维度，因此低维度和高维度的优势都能够兼顾；

2.1.3 自注意力机制

  • 应用背景：自注意力机制是Transformer的核心，抛弃了CNN和RNN架构，完全以注意力机制构建，不断训练词语之间的关联权重；

  • 数学表达：本质是权重系数的计算方法，其计算公式为；

    • Q(Query)：查询向量，表示当前正在处理的词；与所有K进行匹配，以计算当前词对其他词的关注程度；
    • K(Key)：键向量，表示序列中所有的词；与Q作相似度计算，功能和Q相似；
    • V(Value)：值向量，表示序列中所有词的实际信息；与得到的注意力权重做加权求和，得到最终结果；
    • dk：查询维度，维度等于Q和K的维度长度；用于缩放点积结果，防止softmax梯度消失(取根号是为了归一化；

  • 图解公式：以向量图和矩阵图解释自注意力机制的工作原理；

    • QK匹配：如上图所示，q1与其他所有k即k1、k2、k3和k4都进行运算并经过softmax归一化后得到权重得分，从而知道该词与其他词的关联程度；而在矩阵图中k1、k2、k3和k4都可以表示在同一个矩阵K上，发挥现在GPU并行计算的能力；
      • 注意力机制的原理：q向量和k向量的点积公式为qkcos，所以两个越是相似，点积值越大(因为夹角小cos越大)
    • 加权给V：由QK匹配得到的权重得分a11等值与v1、v2、v3和v4点积后可得到该词的新向量，新向量更好的表达了该词与其他词的关联程度；而在矩阵图中，v1、v2、v3和v4都可以表示在同一个矩阵V上；

  • 应用背景：掩码自注意力机制的提出主要解决两个问题：问题一是输入序列之间长度不一致，问题二是Transformer第二个输入头在预测下一个字时需要遮蔽未来的信息；

  • 数学表达：其数学本质是自注意力机制上增加一个Mask，如下式所示；

  • 问题一：以矩阵图解释第一个问题的掩码自注意力机制的工作原理；

    • 遮蔽差值：存在两个长度不同的输入序列时，对短句子缺失的词做遮蔽处理；

  • 问题二：以矩阵图解释第二个问题的掩码自注意力机制的工作原理；

    • 遮蔽未来：解码器负责重要的预测功能，如果将未来的所有信息(QK的右上角)都告知，那么预测就没有意义；因此，此处在QK计算完成后在追加一个Mask，这个Mask的右上角无限接近于0，从而实现遮蔽全部未来信息的功能；

  • 应用背景：Transformer是编码器-解码器结构，需要通过交叉注意力机制交换两边的输入信息，其实本质就是自注意力机制，只是输入信息由编码器和解码器两边构成；

三. RNN时序神经网络

3.1 RNN算法详解

3.1.1 循环单元

  • 应用背景：人工神经网络的结构为单向结构，无法处理包含时间信息的输入数据，因此RNN提出循环单元充分上一个时间的输出，从而使RNN能够处理含时间信息的数据；

  • 数学表达：本质是在神经网络上增加上一个时间的输出； ht = f(Whhht − 1 + Wxhxt + bh)

    • 神经网络：Wxh和bh是输入到隐藏状态的权重矩阵和偏置项，f则是增强非线性表达能力的激活函数；
    • 上一时刻的加入：Whh是上一次结构隐藏状态到本次结构隐藏状态的权重矩阵，ht-1是上一个时间输入后的输出，ht则是这个时间输入的输出；

  • 图解公式：以结构图直接解释；

    • 神经网络：输入Xt经神经元即线性变换和激活函数后输出ht，属于标准的神经元结构；
    • 上一时刻的加入：输入Xt-1得到的ht-1会以一定比重(权重矩阵)与Xt融合；

3.2 LSTM算法详解

3.2.1 LSTM单元

  • 应用背景：RNN只能结合相邻节点的特征，因此长程依赖问题显著；另一方面，RNN的简易结构存在梯度消失和梯度爆炸的风险。因此，LSTM应运而生；

  • 图解公式：本处介绍结合LSTM单元的公式和结构图，LSTM相对RNN做出了多处改进；

    • 遗忘门：遗忘门决定是否保留，其公式为 ft = σ(Wf ⋅ [ht − 1, Xt] + bf)

      • 记忆拼接：上一次的记忆数据ht-1和本次输入Xt拼接并进行线性变换；
      • 筛选记忆：因为sigmoid函数后会得到0和1之间的随机数，0代表忘记上一次的数据，1则代表保留；值得注意的是，sigmoid后作用于ct-1上，所以此处的遗忘门主要是希望遗忘细胞C历史信息中部分无用的信息；

    • 输入门：输入门则是确定输入信息，其公式为

      • 二次筛选：it和遗忘门结构一样，只是权重不同；在功能方面，此处sigmoid作用于ct，所以此处输入门是希望保留候选细胞状态Ct部分有用信息并写入(注意是部分，如果是完全则和遗忘门完全一样了)；
      • 标准输入：Ct是标准的神经元输出，与it乘积后再与遗忘门输出相加；

    • 细胞状态更新：更新Ct

      • 更新Ct：将遗忘门和输入门的输出求和得到新的Ct值；

    • 输出门：输出结果，并将这一次的状态传递给下一次

      • 三次筛选：第三路sigmoid的作用是最终结果Ct做筛选，将Ct的有用信息输出，从而得到ht;

      • 更新参数：至此ht和Ct均以更新完成；

3.3 GRU算法详解

3.3.1 GRU单元

  • 应用背景：RNN 只能结合相邻节点的特征，因此长程依赖问题显著；另一方面，RNN 的简易结构存在梯度消失和梯度爆炸的风险。LSTM 虽解决了该问题，但结构复杂、计算开销大。因此，GRU（Gated Recurrent Unit） 应运而生，它在保留门控机制的同时，简化了结构、提升了训练效率，是 LSTM 的经典轻量化变体。

  • 图解公式：本处介绍结合 GRU 单元的公式和结构图，GRU 相对 RNN、LSTM 做出了结构简化与核心改进

    • 重置门：重置门决定如何融合新输入与历史记忆，其公式为； rt = σ(Wr ⋅ [ht − 1, xt] + br)

      • 记忆拼接：上一次的隐藏状态ht−1和本次输入xt拼接后进行线性变换；
      • 重置控制：sigmoid 函数输出 0~1，0 代表完全丢弃历史记忆，1 代表完全保留历史记忆；与LSTM遗忘门不同的是，这里sigmoid直接作用于ht-1，没有细胞ct-1，所以此处的重置门用于控制 “历史信息对当前状态的影响程度”。

    • 更新门：更新门是 GRU 的核心，同时承担 “遗忘历史” 和 “写入新信息” 的双重功能（替代了 LSTM 的遗忘门 + 输入门），其公式为； zt = σ(Wz ⋅ [ht − 1, xt] + bz)

      • 双重控制：zt 输出 0~1，zt 越接近 1，代表保留越多历史信息；越接近 0，代表写入越多新信息；
      • 结构简化：相比 LSTM 的两个门控，GRU 用一个更新门同时完成 “遗忘旧记忆” 和 “更新新记忆”，大幅减少了参数数量和计算量。

    • 候选隐藏状态：候选隐藏状态是待写入的新记忆信息，其公式为； h̃t = tanh (W ⋅ [rt ⊙ ht − 1, xt] + b)

      • 重置门作用：用重置门rt与历史状态ht−1逐元素相乘（⊙），过滤掉无关的历史信息，仅保留与当前输入相关的记忆；
      • 新记忆生成：通过 tanh 函数生成 - 1~1 的候选新状态，作为当前时刻的待写入信息。

    • 最终隐藏状态更新：最终隐藏状态由更新门加权融合历史状态与候选状态得到，公式为； ht = (1 − zt) ⊙ ht − 1 + zt ⊙ h̃t

      • 加权融合：(1−zt) 控制对历史状态ht−1的保留比例，zt 控制对新候选状态h~t的写入比例；
      • 无独立细胞状态：GRU 取消了 LSTM 的独立细胞状态ct，直接用隐藏状态ht承载记忆，进一步简化了结构，训练速度更快。
